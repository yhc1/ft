互動式資料視覺化
利用 AI 分析與整理大型資料集，並強化視覺化敘事，讓內容更具吸引力，觸及更廣泛的讀者。
Case Study
伊波拉病毒熱點
建立特徵時需要對病毒傳播有足夠的認識。
可能會有 validation 的問題，因為早期預測注定資料樣本點不足，搭配專家驗證能提升信心水準。
因此模型需具備足夠的解釋力與透明度，邏輯回歸與樹模型都是相對適合的方法。
分析選舉期間社群媒體對政治人物的情緒
搜集社群媒體貼文需進行 spam filter，每篇文章對應到不同人物時也需注意歸屬問題。若不要求太高的覆蓋率，採用白名單的做法會簡單不少。
編碼整篇文章會比使用詞向量（Word2Vec）更好，能捕捉語意上的細微差異，後處理時再進行分詞即可。
General
Metrics
Engagement：假設豐富圖文的內容能提升使用者的點擊意願與停留時間，搭配推薦深入的相關報導也能進一步提升 engagement。
Technical Assessment
Sentiment Analysis 中若要相對準確的預測，則對模型選擇的挑戰較高，包含以下
Evaluation set 的建立：需要建立一組標準的測試集，收錄了常見案例以及邊緣案例並人工標注，才能夠去評估不同模型的準確度。
Model selection: 情緒模型通常建立在預先定義的資料集中，比如推文，語境的不同會很大的影響模型效能，另外情緒模型會有預先定義的標籤（如開心、生氣等）這不一定符合需求。一個比較建議的方式是用 LLM 進行分類，要分析的數項在 Millions 層級底下花費都不會太貴。
預測性洞察高度依賴領域知識，在各階段都領域專家的深度參與
資料搜集：什麼樣的資料有助於判斷，比如在傳染病的預測中根據傳染的方式不同可能就需要不同的數據。
特徵建立：在預測時尤其在較小的資料量中，注入先驗知識幫助模型更快擬合是很實際的做法（e.g. 有效的約束參數空間），比如近期的資料權重更高、兩個不同的資料進行組合如 CTR 是有代表性的。
效果評估與校正：在衡量不同模型帶來的結果時，有能力去基於結果進行數據與模型的校正。
使用生成式模型能降低對於專家的依賴，可以根據過往類似的事件設計 distribution 的參數。
響應式設計與效能，臺灣讀者大量使用手機閱讀新聞，互動式視覺化在行動裝置上的操作體驗與載入速度是需要面對的實際問題。
